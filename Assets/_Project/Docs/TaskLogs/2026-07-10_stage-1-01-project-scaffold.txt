Этап roadmap: Этап 1 - основа Unity-проекта и взаимодействия.
Задача: 1.1 - создать Unity-проект и архитектурный каркас Clean Architecture.

Что сделано:
- Создан Unity-проект на Unity 6.5.0f1.
- Добавлены базовые Unity-пакеты: Input System, URP, AI Navigation, UI и Test Framework.
- Создана структура Assets/_Project/ для Core, Application, Infrastructure, Presentation, Tests, Content, Scenes, Settings и Docs.
- Созданы assembly definitions, задающие направление зависимостей Clean Architecture.
- Core, Application и Infrastructure не имеют UnityEngine references.
- Создана живая developer-карта файлов в Assets/_Project/Docs/Developer/FileMap.md.
- Подготовлены пустые папки для готовых FBX assets и остальных типов контента. Никакие модели и анимации не добавлялись.

Изменённые файлы и папки:
- Packages/manifest.json
- ProjectSettings/ProjectVersion.txt
- Assets/_Project/Core/HorseParking.Core.asmdef
- Assets/_Project/Application/HorseParking.Application.asmdef
- Assets/_Project/Infrastructure/HorseParking.Infrastructure.asmdef
- Assets/_Project/Presentation/HorseParking.Presentation.asmdef
- Assets/_Project/Tests/HorseParking.Core.Tests.asmdef
- Assets/_Project/Content/
- Assets/_Project/Scenes/
- Assets/_Project/Settings/
- Assets/_Project/Docs/Developer/FileMap.md

Проверка:
- Unity Editor найден: 6000.5.0f1.
- Проект создан; версия редактора записана в ProjectSettings/ProjectVersion.txt.
- Unity успешно открыл проект в batch mode и создал meta-файлы для структуры Assets/_Project/.
- Новые пакеты из Packages/manifest.json будут окончательно разрешены Package Manager при следующем полном открытии проекта в Unity Editor; текущий packages-lock.json пока содержит только исходные встроенные пакеты.

Ограничения:
- Игровая сцена, управление, камера и взаимодействие ещё не реализованы.
- Assets отсутствуют намеренно: по правилам проекта они добавляются только как готовые файлы, предоставленные пользователем.
