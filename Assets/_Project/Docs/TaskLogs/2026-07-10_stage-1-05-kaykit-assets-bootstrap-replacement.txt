Этап roadmap: Этап 1 - основа Unity-проекта и взаимодействия.
Задача: 1.5 - заменить видимые Unity-примитивы Bootstrap-сцены готовыми KayKit FBX assets.

Что сделано:
- Пользователь добавил KayKit Medieval Hexagon Pack в проект.
- Исправлено имя папки Enviroment на Environment.
- BootstrapSceneBuilder теперь создаёт землю только из готового KayKit hex_grass.fbx: сетка 7x7 из 49 FBX tiles.
- BootstrapSceneBuilder теперь создаёт interaction target из готового KayKit building_scaffolding.fbx.
- Unity Plane и Unity Cube удалены из builder-логики.
- Для готовых FBX mesh добавляются только невидимые MeshCollider-компоненты, необходимые для ходьбы и raycast-взаимодействия.
- Player interactor теперь корректно находит IInteractionTarget на родительском объекте collider-а.
- Обновлены карта файлов и инструкция запуска.

Изменённые файлы:
- Assets/_Project/Presentation/Editor/BootstrapSceneBuilder.cs
- Assets/_Project/Presentation/Player/FirstPersonPlayerController.cs
- Assets/_Project/Docs/Developer/FileMap.md

Проверка:
- Unity batch-проверка компиляции пройдена без ошибок.
- BootstrapSceneBuilder выполнен без ошибок; Bootstrap.unity пересобрана.
- В сцене проверено: 49 объектов ParkingGroundTile_ReadyFBX, 1 объект ConstructionScaffolding_ReadyFBX_InteractionTarget, старые имена Unity Plane/Cube отсутствуют.
- Оставшаяся пустая папка Enviroment с дублирующими .meta-файлами безопасно удалена после проверки переноса всех не-meta файлов.

Исправление визуальной проблемы:
- Сетка hex_grass.fbx визуально не принята: пакет изометрический и не подходит для основы first-person парковки.
- Сетка удалена из BootstrapSceneBuilder и заменена одним масштабированным готовым FBX building_dirt.fbx из того же KayKit-набора.
- Исправление не меняет Core, DI, камеру или взаимодействие; после него roadmap продолжается с Этапа 1.
- Bootstrap.unity должна быть пересобрана командой `Horse Parking -> Build Bootstrap Scene` в открытом Unity Editor: внешняя batch-пересборка не заменяет сцену, пока она открыта пользователем.
- После закрытия Unity пользователем сцена успешно пересобрана в batch mode. Проверка YAML: старых ParkingGroundTile_ReadyFBX = 0, нового ParkingGround_ReadyFBX_Dirt = 1, ConstructionScaffolding_ReadyFBX_InteractionTarget = 1.

Откат по решению пользователя:
- Визуальная сборка KayKit признана не соответствующей ТЗ: она не реализует парковочную, строительную, логистическую или экономическую петлю.
- Из BootstrapSceneBuilder удалены ground и scaffolding instances; удалён DebugInteractionTarget.
- KayKit-набор не удалён и сохранён в Content для возможного точечного использования в будущем.
- Bootstrap возвращён к роли технического фундамента Этапа 1: Composition Root, камера и контроллер без ложной игровой локации.
